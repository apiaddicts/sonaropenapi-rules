Depending of the verb, some path structures are not encouraged. This can be a hint of a design flaw.

Allowed paths

VERB STRUCTURE
GET /resources
GET /resources/{id}
GET /resources/{id}/subresources
GET /resources/me
POST /resources
POST /resources/get
POST /resources/delete
POST /resources/{id}/subresources
PUT /resources/{id}
PUT /resources/me
PATCH /resources/{id}
PATCH /resources/me
DELETE /resources/{id}
DELETE /resources/me

Noncompliant Code Example (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /customers/{customerId}:
    post: # Noncompliant {{OAR018: Operation not recommended for resource path: POST /customers/{customerId}}
      parameters:
      - name: customerId
        in: path
        required: true
        type: integer
      - name: body
        in: body
        required: true
        schema:
          type: object
      responses:
        200:
          description: Ok

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /customers/{customerId}:
    put:
      parameters:
      - name: customerId
        in: path
        required: true
        type: integer
      - name: body
        in: body
        required: true
        schema:
          type: object
      responses:
        200:
          description: Ok
