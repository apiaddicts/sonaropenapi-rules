It is validated that each route alternates its parts between static and variable parts, starting with the static parts.

Examples:

  • /{id} -> Error
  • /facturas/{id}/{lineId} -> Error
  • /facturas/{id} -> Ok
  • /facturas/get -> Ok
  • /facturas/{id}/lineas/{lineId} -> Ok
  • /facturas/{id}/lineas/get -> Ok

Noncompliant Code Example (OpenAPI 2)

All the examples are in YAML

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /customers/{customerId}/{invoiceId}:
    get:
      parameters:
        - in: path
          name: customerId
          type: integer
          required: true
        - in: path
          name: invoiceId
          type: integer
          required: true
      responses:
        200:
          description: Ok

Compliant Solution (OpenAPI 2)

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /customers/{customerId}/invoices:
    get:
      parameters:
        - in: path
          name: customerId
          type: integer
          required: true

        200:

    get:
