Depending of the verb, some path structures are not encouraged. This can be a hint of a design flaw.

Allowed paths

VERB STRUCTURE
GET /resources
GET /resources/{id}
GET /resources/{id}/subresources
POST /resources
POST /resources/get
POST /resources/delete
POST /resources/{id}/subresources
PUT /resources/{id}
PATCH /resources/{id}
DELETE /resources/{id}

Noncompliant Code Example (OpenAPI 2)

YAML

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /resources:
    get:
      responses:
        200:
          description: Ok
    post:
      responses:
        200:
          description: Ok
    put: # Noncompliant {{OAR018: Operation not recommended for resource path:  /resources}}
      responses:
        200:
          description: Ok
    patch: # Noncompliant {{OAR018: Operation not recommended for resource path:  /resources}}
      responses:
        200:
          description: Ok
    delete: # Noncompliant {{OAR018: Operation not recommended for resource path:  /resources}}
      responses:
        200:
          description: Ok

Compliant Solution (OpenAPI 2)

YAML

swagger: "2.0"
info:
  title: Swagger Petstore
  version: 1.0.0
paths:
  /resources/{r_id}:
    get:
      responses:
        200:
          description: Ok
    put:
      responses:
        200:
          description: Ok
    patch:
      responses:
        200:
          description: Ok
    delete:
      responses:
        200:
          description: Ok
