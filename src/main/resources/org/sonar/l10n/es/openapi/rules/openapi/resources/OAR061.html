Los códigos de respuesta en las operaciones GET deben definirse de acuerdo con el estándar

Ejemplo de código no compatible (OpenAPI 2)

Todos los ejemplos están en YAML

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        default:
          description: Default
    post:
      responses:
        400:
          description: Bad request

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        200:
          description: OK
        202:
          description: Accepted
        206:
          description: Partial collection
