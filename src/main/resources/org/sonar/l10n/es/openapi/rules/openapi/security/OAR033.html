Anular ciertos encabezados o permitir que se establezca cualquier encabezado y no especificar los encabezados requeridos puede causar algunas vulnerabilidades en la API.

Ejemplo de código no compatible (OpenAPI 2)

Todos los ejemplos están en YAML

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      parameters:
        - in: header
          name: Accept
          type: string
          required: true
        - in: header
          name: Authorization
          type: string
          required: true
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      parameters:
        - in: header
          name: x-api-key
          type: string
          required: true
        - in: header
          name: traceId
          type: string
          required: true
        - in: header
          name: dateTime
          type: string
          required: true
        - in: query
          name: Authorization
          type: string
          required: true
      responses:
        200:
          description: Ok

Ejemplo de código no compatible (OpenAPI 3)

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
components:
  parameters:
    userId:
      in: header
      name: x-user-id
      schema:
        type: string
      required: true
paths:
  /pets:
    get:
      parameters:
      - $ref: '#/components/parameters/userId'
      responses:
        200:
          description: Ok
