NORMA: Definición API - Monitorización

Para facilitar la monitorización de las APIs desde herramientas externas se debe incluir un endpoint de health check en todas ellas. Dicho endpoint informará del estado de salud de la API por medio de la respuesta estándar, estableciendo el valor adecuado en el campo status del objeto result. En caso de error es recomendable incluir la causa del mismo en el array de errores de la respuesta estándar. Este endpoint se definirá siempre de la siguiente manera: GET {basepath}/status

Ejemplo de código no compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    get:
      responses:
        200:
          description: Ok
  /status:
    get:
      responses:
        200:
          description: Ok
