Al definir una API se deben definir cuáles son los formatos de salida de datos.

Según el estándar el formato por defecto será application/json, salvo algunas excepciones.

Ejemplo de código no compatible (OpenAPI 2)

JSON

{
    "swagger": "2.0", 
    "info": {
        "title": "Swagger Petstore", 
        "version": "1.0"
    }, 
    "produces": [
        "application/xml"
    ], 
    "paths": {
        "/pets": {
            "put": {
                "responses": {
                    "200": {
                        "description": "Ok"
                    }
                }
            }
        }
    }
}

YAML

swagger: "2.0"
info:
  title: Swagger Petstore
  version: "1.0"
produces:
  - application/xml
paths:
  /pets:
    put:
      responses:
        200:
          description: Ok

Ejemplo de código no compatible (OpenAPI 3)

JSON

{
    "openapi": "3.0.0", 
    "info": {
        "title": "Swagger Petstore", 
        "version": "1.0"
    }, 
    "paths": {
        "/pets": {
            "post": {
                "responses": {
                    "200": {
                        "description": "Ok", 
                        "content": {
                            "application/xml": {
                                "schema": {
                                    "type": "object"
                                }
                            }
                        }
                    }
                }
            }
        }
    }
}

YAML

openapi: 3.0.0
info:
  title: Swagger Petstore
  version: "1.0"
paths:
  /pets:
    post:
      responses:
        200:
          description: Ok
          content:
            application/xml:
              schema:
                type: object
