El nombre de host debe ser un subdominio del nombre de dominio de la organización.

Ejemplo de código no compatible (OpenAPI 2)

Todos los ejemplos están en YAML

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
host: other.domain.com
paths: {}

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
host: api.apiaddicts.org
paths: {}

Ejemplo de código no compatible (OpenAPI 3)

openapi: "3.0.0"
info:
  version: 1.0.0
  title: Swagger Petstore
servers:
  - url: https://aws-dev-apiaddicts.org/api-users/v1/

  - url: https://aws-apiaddicts.org/api-users/v1/
