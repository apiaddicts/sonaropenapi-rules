Los códigos de respuesta en las operaciones PATCH deben definirse de acuerdo con el estándar

Ejemplo de código no compatible (OpenAPI 2)

Todos los ejemplos están en YAML

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    patch:
      responses:
        400:
          description: Bad request

Solución compatible (OpenAPI 2)

swagger: "2.0"
info:
  version: 1.0.0
  title: Swagger Petstore
paths:
  /pets:
    patch:
      responses:
        200:
          description: OK

Ejemplo de código no compatible (OpenAPI 3)

openapi: "3.0.0"
info:

paths:

      responses:
